Git基础

Git基础

1、创建版本库（仓库）

mkdir Repository

2、初始化

git init

3、添加文件到仓库

git add 文件

4、提交文件到仓库（版本库） -m 后是提交说明，方便管理

git commit -m ""

5、Git版本库（仓库）状态

git status

6、查看改动内容

git diff

7、查看提交时间线（提交历史日志） –pretty=oneline 显示格式，一行显示

git log --pretty=oneline

8、查看命令历史

git reflog

9、回退上一版本

git reset --hard HEAD^

10、恢复到指定版本

git reset --hard 版本号

11、查看版本库中最新版本和工作区中文件的区别

git diff HEAD -- 文件

12、Git暂存区（stage|index）和工作区（Working Directory）

.git是Git的版本库其中包含暂存区（stage|index) 和初始化时自动创建的master分支 ，master 分支中 包含 指针head；
git add添加文件到暂存区；
git commit提交暂存区文件到Git版本库分支；
未添加到暂存区的文件无法提交到版本库分支之中，可以使用git diff head – 文件查看版本库最新版本文件和工作区中文件的区别。

13、撤销工作区文件修改

git checkout -- 文件

14、撤销添加到暂存区的文件到工作区同时撤销文件修改

14.1 撤销暂存区文件到工作区

git reset HEAD 文件

14.2 撤销工作区文件修改

git checkout -- 文件

15、删除版本库中的文件

git rm 文件

16、恢复误删除文件，从版本库中替换工作区文件 （前提条件是版本库中的文件未删除，并且只能恢复版本库中最新版本文件）

git checkout -- 文件

17、本地Git版本库关联GitHub远程库

git remote add origin git@github.com:GitHubName/gitDirectory

18、首次推送本地版本库到远程库(-u是为了关联本地master分支和远程库中的master分支，简化之后推送和拉取命令)

git push -u origin master

19、推送本地库修改到远程库

git push origin master

20、查看远程库状态

git temote -v

21、删除远程库（解除本地仓库和远程库的关联）

git temote rm orgin

22、创建并切换分支

//1.创建分支dev
git branch dev
//2.切换分支到dev
git checkout dev
//上面的两个步骤可以合并为一个
git checkout -b dev

23、查看当前分支

git branch

24、合并指定分支到当前分支

/*和并分支dev到master*/
//1.切换分支到master
git checkout master
//2.合并dev到master
git merge dev

25、删除分支dev

git branch -d dev
